## Conversa

Durante o planejamento colaborativo, os participantes devem poder expressar sua opinião sobre atividades, atrações, hospedagens, eventos e demais opções cadastradas na viagem. Para isso, o sistema deve permitir registrar avaliações positivas ou negativas em cada opção disponível. Essas avaliações ficarão visíveis para todos os participantes e servirão como apoio para as decisões do grupo.

## Critérios de Aceitação

| #   | Critério                                                                        |
| --- | ------------------------------------------------------------------------------- |
| 1   | O sistema deve permitir que participantes visualizem as avaliações de uma opção |
| 2   | O sistema deve permitir registrar uma avaliação positiva                        |
| 3   | O sistema deve permitir registrar uma avaliação negativa                        |
| 4   | O sistema deve permitir apenas uma avaliação por participante em cada opção     |
| 5   | O participante deve poder alterar sua avaliação posteriormente                  |
| 6   | O sistema deve atualizar automaticamente a contagem de avaliações               |
| 7   | O sistema deve exibir a quantidade de avaliações positivas                      |
| 8   | O sistema deve exibir a quantidade de avaliações negativas                      |
| 9   | Apenas participantes vinculados à viagem podem avaliar opções                   |
| 10  | O sistema deve armazenar o histórico das avaliações realizadas                  |
| 11  | O sistema deve impedir avaliações em viagens encerradas                         |

## Cenários de Teste

| Cenário                     | Dado                                               | Quando                                 | Então                                                             |
| --------------------------- | -------------------------------------------------- | -------------------------------------- | ----------------------------------------------------------------- |
| Avaliação positiva          | O participante visualiza uma opção da viagem       | Seleciona a reação positiva            | O sistema registra a avaliação e atualiza a contagem              |
| Avaliação negativa          | O participante visualiza uma opção da viagem       | Seleciona a reação negativa            | O sistema registra a avaliação e atualiza a contagem              |
| Alteração de avaliação      | O participante já avaliou uma opção                | Seleciona a reação oposta              | O sistema substitui a avaliação anterior                          |
| Avaliação duplicada         | O participante já registrou uma avaliação positiva | Seleciona novamente a mesma reação     | O sistema mantém apenas uma avaliação                             |
| Usuário sem acesso à viagem | Um usuário não participante acessa uma opção       | Tenta registrar uma avaliação          | O sistema impede a operação                                       |
| Viagem encerrada            | A viagem está encerrada                            | O participante tenta avaliar uma opção | O sistema bloqueia a avaliação                                    |
| Atualização das contagens   | Uma avaliação é registrada                         | A operação é concluída                 | O sistema atualiza os totais exibidos para todos os participantes |

## Observações

1. As avaliações podem ser utilizadas para apoiar decisões do grupo durante o planejamento.

2. Uma avaliação positiva representa interesse ou aprovação da opção.

3. Uma avaliação negativa representa desinteresse ou reprovação da opção.

4. Cada participante pode possuir apenas uma avaliação ativa por opção.

5. O sistema deve exibir as quantidades de avaliações positivas e negativas de forma visível para todos os participantes.

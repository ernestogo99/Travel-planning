| -------- | ----------------------------------------------------------------------- |
| Como     | Organizador da viagem                                                   |
| Quero    | Criar uma nova viagem informando minhas informações básicas             |
| Para que | Possa iniciar o planejamento e organizar os detalhes da viagem em grupo |

## Conversa

O Organizador da viagem precisa criar uma viagem para iniciar o planejamento colaborativo. Durante a criação, devem ser informadas as informações iniciais da viagem, como destino, período e orçamento estimado. Após a criação, a viagem ficará disponível para que o Organizador possa adicionar participantes e iniciar o planejamento com o grupo.

## Critérios de Aceitação

| #   | Critério                                                                                                  |
| --- | --------------------------------------------------------------------------------------------------------- |
| 1   | O sistema deve disponibilizar uma opção para criação de uma nova viagem                                   |
| 2   | O sistema deve disponibilizar um formulário contendo as informações necessárias para criação da viagem    |
| 3   | O formulário deve permitir informar o nome da viagem                                                      |
| 4   | O formulário deve permitir informar o destino da viagem                                                   |
| 5   | O formulário deve permitir informar a data inicial e final da viagem                                      |
| 6   | O formulário deve permitir informar um orçamento estimado para a viagem                                   |
| 7   | O sistema deve validar que os campos obrigatórios sejam preenchidos antes de permitir a criação da viagem |
| 8   | O sistema deve validar que a data final da viagem seja posterior à data inicial                           |
| 9   | O sistema deve gerar um identificador único para cada viagem criada                                       |
| 10  | O sistema deve associar automaticamente o usuário criador como Organizador da viagem                      |
| 11  | O sistema deve armazenar as informações da viagem após a confirmação da criação                           |
| 12  | O sistema deve exibir uma mensagem de confirmação após a criação bem-sucedida                             |
| 13  | O sistema deve direcionar o Organizador para a página de gerenciamento da viagem criada                   |

## Cenários de Teste

| Cenário                       | Dado                                                        | Quando                                                                             | Então                                                                                  |
| ----------------------------- | ----------------------------------------------------------- | ---------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| Criação de viagem válida      | O Organizador acessa a tela de criação de viagem            | Preenche todos os campos obrigatórios com informações válidas e confirma a criação | O sistema cria a viagem, exibe mensagem de sucesso e direciona para a página da viagem |
| Nome da viagem vazio          | O Organizador acessa a tela de criação                      | Deixa o campo nome vazio e tenta criar a viagem                                    | O sistema informa que o campo nome é obrigatório                                       |
| Destino vazio                 | O Organizador acessa a tela de criação                      | Não informa o destino e tenta criar a viagem                                       | O sistema informa que o destino é obrigatório                                          |
| Data inicial vazia            | O Organizador acessa a tela de criação                      | Não informa a data inicial                                                         | O sistema informa que a data inicial é obrigatória                                     |
| Data final anterior à inicial | O Organizador informa datas inválidas                       | Define uma data final anterior à data inicial                                      | O sistema exibe uma mensagem informando que o período da viagem é inválido             |
| Orçamento não informado       | O Organizador cria uma viagem sem informar orçamento        | Confirma a criação                                                                 | O sistema permite a criação caso o orçamento seja opcional                             |
| Falha ao salvar viagem        | O Organizador preenche dados válidos                        | Ocorre erro durante o armazenamento                                                | O sistema informa que não foi possível criar a viagem                                  |
| Cancelamento da criação       | O Organizador acessa a tela de criação e preenche os campos | Seleciona a opção cancelar                                                         | O sistema retorna para a tela anterior sem salvar a viagem                             |
| Criação de múltiplas viagens  | O Organizador já possui uma viagem criada                   | Inicia uma nova criação                                                            | O sistema permite criar uma nova viagem independente da anterior                       |

## Observações

1. Informações da viagem:
   - Nome da viagem: identificação definida pelo Organizador
   - Destino: local principal da viagem
   - Data inicial: início previsto da viagem
   - Data final: término previsto da viagem
   - Orçamento: valor estimado disponível para planejamento

2. Apenas usuários autenticados podem criar viagens.

3. O usuário que cria a viagem recebe automaticamente o papel de Organizador, podendo posteriormente convidar participantes e gerenciar decisões do grupo.

| Para que | Eu possa iniciar o planejamento colaborativo da viagem em grupo |

## Conversa

O Organizador da viagem precisa criar uma nova viagem para iniciar o planejamento colaborativo. Durante a criação, o sistema deve permitir informar os dados básicos da viagem, como objetivo, localização, período e imagem de identificação. Após a criação, a viagem ficará disponível para gerenciamento, permitindo posteriormente a inclusão de participantes e a realização do planejamento em grupo.

## Critérios de Aceitação

| #   | Critério                                                                                               |
| --- | ------------------------------------------------------------------------------------------------------ |
| 1   | O sistema deve disponibilizar uma opção para criação de uma nova viagem                                |
| 2   | O sistema deve disponibilizar um formulário contendo as informações necessárias para criação da viagem |
| 3   | O formulário deve permitir informar o objetivo da viagem                                               |
| 4   | O formulário deve permitir informar a cidade da viagem                                                 |
| 5   | O formulário deve permitir informar o logradouro da viagem                                             |
| 6   | O formulário deve permitir informar o número do endereço                                               |
| 7   | O formulário deve permitir informar a unidade federativa (UF)                                          |
| 8   | O formulário deve permitir informar a data inicial da viagem                                           |
| 9   | O formulário deve permitir informar a data final da viagem                                             |
| 10  | O formulário deve permitir adicionar uma imagem representativa da viagem                               |
| 11  | O sistema deve validar que os campos obrigatórios sejam preenchidos antes da criação                   |
| 12  | O sistema deve validar que a data final seja posterior à data inicial                                  |
| 13  | O sistema deve gerar um identificador único para a viagem                                              |
| 14  | O sistema deve associar automaticamente o usuário criador como Organizador da viagem                   |
| 15  | O sistema deve armazenar as informações da viagem após a confirmação da criação                        |
| 16  | O sistema deve exibir uma mensagem de confirmação após a criação bem-sucedida                          |
| 17  | O sistema deve direcionar o Organizador para a página principal da viagem criada                       |

## Cenários de Teste

| Cenário                      | Dado                                                    | Quando                                                     | Então                                                     |
| ---------------------------- | ------------------------------------------------------- | ---------------------------------------------------------- | --------------------------------------------------------- |
| Criação de viagem válida     | O Organizador acessa a tela de criação de viagem        | Preenche todos os campos obrigatórios e confirma a criação | O sistema cria a viagem e exibe mensagem de sucesso       |
| Objetivo da viagem vazio     | O Organizador acessa a tela de criação                  | Não informa o objetivo da viagem                           | O sistema informa que o campo é obrigatório               |
| Cidade não informada         | O Organizador acessa a tela de criação                  | Não informa a cidade da viagem                             | O sistema informa que o campo é obrigatório               |
| Data final inválida          | O Organizador informa uma data final anterior à inicial | Confirma a criação                                         | O sistema informa que o período da viagem é inválido      |
| Inclusão de imagem           | O Organizador seleciona uma imagem válida               | Confirma a criação da viagem                               | O sistema associa a imagem à viagem                       |
| Falha ao salvar viagem       | O Organizador informa dados válidos                     | Ocorre erro durante o armazenamento                        | O sistema informa que não foi possível criar a viagem     |
| Cancelamento da criação      | O Organizador preenche parte das informações            | Seleciona a opção cancelar                                 | O sistema retorna para a tela anterior sem criar a viagem |
| Criação de múltiplas viagens | O Organizador já possui viagens cadastradas             | Cria uma nova viagem                                       | O sistema permite a criação normalmente                   |

## Observações

1. Informações iniciais da viagem:
   - Objetivo da viagem
   - Cidade
   - Endereço
   - Número
   - Unidade Federativa (UF)
   - Data inicial
   - Data final
   - Imagem da viagem

2. Apenas usuários autenticados podem criar viagens.

3. O usuário que cria a viagem recebe automaticamente o papel de Organizador.

4. A imagem da viagem será utilizada para facilitar sua identificação visual dentro da plataforma.

5. Após a criação, o Organizador poderá convidar participantes e iniciar o planejamento colaborativo.

## Cartão

| Elemento | Descrição                                                                    |
| -------- | ---------------------------------------------------------------------------- |
| Como     | Usuário autenticado                                                          |
| Quero    | Visualizar minhas viagens e planejamentos                                    |
| Para que | Eu possa acompanhar rapidamente as viagens das quais participo ou administro |

## Conversa

O usuário precisa acessar um painel inicial que apresente um resumo das viagens e planejamentos vinculados à sua conta. O sistema deve exibir as viagens administradas pelo usuário, os planejamentos dos quais ele participa e as atividades recentemente acessadas. A partir desse painel, o usuário deve conseguir navegar rapidamente para uma viagem específica ou iniciar um novo planejamento.

## Critérios de Aceitação

| #   | Critério                                                                                                            |
| --- | ------------------------------------------------------------------------------------------------------------------- |
| 1   | O sistema deve disponibilizar uma página inicial de viagens após a autenticação                                     |
| 2   | O sistema deve exibir a seção "Minhas viagens"                                                                      |
| 3   | O sistema deve exibir a seção "Meus planejamentos"                                                                  |
| 4   | O sistema deve exibir apenas viagens associadas ao usuário autenticado                                              |
| 5   | Cada viagem deve exibir seu nome                                                                                    |
| 6   | Cada viagem deve exibir sua localização principal                                                                   |
| 7   | Cada viagem deve exibir os participantes associados                                                                 |
| 8   | O sistema deve permitir selecionar uma viagem para visualizar seus detalhes                                         |
| 9   | O sistema deve disponibilizar a opção "Ver mais" quando existirem mais viagens do que o limite exibido inicialmente |
| 10  | O sistema deve exibir a seção "Últimas atividades acessadas"                                                        |
| 11  | O sistema deve permitir acessar uma atividade exibida na lista de atividades recentes                               |
| 12  | O sistema deve disponibilizar a opção "Criar Planejamento"                                                          |
| 13  | Ao selecionar "Criar Planejamento", o sistema deve direcionar o usuário para o fluxo de criação de planejamento     |
| 14  | O sistema deve exibir mensagem apropriada quando não existirem viagens ou planejamentos cadastrados                 |
| 15  | O sistema deve atualizar as informações exibidas sempre que houver alterações nas viagens associadas ao usuário     |

## Cenários de Teste

| Cenário                        | Dado                                                  | Quando                         | Então                                                      |
| ------------------------------ | ----------------------------------------------------- | ------------------------------ | ---------------------------------------------------------- |
| Visualizar viagens existentes  | O usuário possui viagens cadastradas                  | Acessa o painel inicial        | O sistema exibe as viagens associadas ao usuário           |
| Visualizar planejamentos       | O usuário participa de planejamentos                  | Acessa o painel inicial        | O sistema exibe os planejamentos vinculados ao usuário     |
| Acessar viagem                 | O usuário visualiza uma viagem na lista               | Seleciona a viagem             | O sistema abre os detalhes da viagem                       |
| Ver mais viagens               | O usuário possui mais viagens do que o limite exibido | Seleciona "Ver mais"           | O sistema exibe a lista completa de viagens                |
| Criar planejamento             | O usuário acessa o painel inicial                     | Seleciona "Criar Planejamento" | O sistema direciona para a tela de criação de planejamento |
| Visualizar atividades recentes | O usuário possui atividades acessadas anteriormente   | Acessa o painel inicial        | O sistema exibe as atividades recentes                     |
| Nenhuma viagem cadastrada      | O usuário não possui viagens associadas               | Acessa o painel inicial        | O sistema informa que não existem viagens disponíveis      |
| Falha ao carregar informações  | Ocorre erro na recuperação dos dados                  | O usuário acessa o painel      | O sistema exibe mensagem de erro apropriada                |

## Observações

1. O painel deve funcionar como página inicial do módulo de planejamento de viagens.

2. As seções "Minhas viagens" e "Meus planejamentos" podem representar papéis diferentes do usuário dentro da plataforma.

3. As atividades recentes devem permitir acesso rápido às informações utilizadas com maior frequência.

4. Apenas usuários autenticados podem acessar o painel de viagens.

## Conversa

Durante o planejamento da viagem, pode ser necessário remover participantes que desistiram da viagem, foram adicionados incorretamente ou não fazem mais parte do grupo. O Organizador deve poder visualizar os participantes vinculados à viagem e selecionar aqueles que deseja remover. Após a remoção, o participante perderá acesso às informações e funcionalidades relacionadas à viagem.

## Critérios de Aceitação

| #   | Critério                                                                                 |
| --- | ---------------------------------------------------------------------------------------- |
| 1   | O sistema deve disponibilizar uma lista dos participantes da viagem                      |
| 2   | Apenas o Organizador da viagem pode remover participantes                                |
| 3   | O sistema deve permitir selecionar um participante para remoção                          |
| 4   | O sistema deve solicitar confirmação antes de concluir a remoção                         |
| 5   | O sistema deve remover o vínculo do participante com a viagem após a confirmação         |
| 6   | O participante removido não deve mais acessar informações da viagem                      |
| 7   | O sistema deve atualizar automaticamente a lista de participantes                        |
| 8   | O sistema deve registrar a remoção realizada                                             |
| 9   | O sistema deve exibir mensagem de confirmação após a remoção bem-sucedida                |
| 10  | O sistema deve informar quando não for possível concluir a remoção                       |
| 11  | O sistema não deve permitir que o Organizador remova a si mesmo da viagem                |
| 12  | O sistema deve manter o histórico de atividades já realizadas pelo participante removido |

## Cenários de Teste

| Cenário                                      | Dado                                                | Quando                                         | Então                                                     |
| -------------------------------------------- | --------------------------------------------------- | ---------------------------------------------- | --------------------------------------------------------- |
| Remoção de participante                      | O Organizador visualiza a lista de participantes    | Seleciona um participante e confirma a remoção | O sistema remove o participante da viagem                 |
| Cancelamento da remoção                      | O Organizador seleciona um participante             | Cancela a operação antes da confirmação        | O sistema mantém o participante vinculado à viagem        |
| Participante removido tenta acessar a viagem | Um participante foi removido da viagem              | Tenta acessar informações da viagem            | O sistema nega o acesso                                   |
| Atualização da lista                         | Um participante é removido                          | A remoção é concluída                          | O sistema atualiza a lista de participantes exibida       |
| Tentativa de remover o Organizador           | O Organizador seleciona seu próprio usuário         | Confirma a remoção                             | O sistema impede a operação e informa o motivo            |
| Falha na remoção                             | O Organizador solicita a remoção de um participante | Ocorre erro durante o processamento            | O sistema informa que não foi possível concluir a remoção |

## Observações

1. Apenas participantes que já aceitaram o convite e estão vinculados à viagem podem ser removidos.

2. A remoção não deve apagar registros históricos de interações realizadas pelo participante durante o planejamento.

3. O Organizador permanece responsável pelo gerenciamento dos participantes da viagem.

4. Caso futuramente exista mais de um Organizador, regras adicionais de permissão deverão ser definidas.

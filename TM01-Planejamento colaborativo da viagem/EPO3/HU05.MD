## Conversa

Após criar uma viagem, o Organizador deve poder definir e atualizar as informações iniciais que servirão de base para o planejamento colaborativo. Essas informações incluem objetivo da viagem, localização, período da viagem e uma imagem de identificação. Os dados cadastrados devem ficar disponíveis para consulta pelos participantes e poderão ser utilizados em funcionalidades futuras, como recomendações da IA e geração do cronograma.

| 3   | O sistema deve permitir informar a cidade da viagem                                      |
| 4   | O sistema deve permitir informar o logradouro da viagem                                  |

| 8   | O sistema deve permitir informar a data de término da viagem                             |
| 9   | O sistema deve permitir enviar uma imagem representativa da viagem                       |
| 10  | O sistema deve validar que a data final seja posterior à data inicial                    |
| 11  | O sistema deve armazenar as alterações realizadas pelo Organizador                       |
| 12  | O sistema deve exibir mensagem de confirmação após a atualização das informações         |
| 13  | O sistema deve disponibilizar as informações atualizadas para os participantes da viagem |

## Cenários de Teste

| Cenário                               | Dado                                                    | Quando                           | Então                                                     |
| ------------------------------------- | ------------------------------------------------------- | -------------------------------- | --------------------------------------------------------- |
| Atualização das informações da viagem | O Organizador acessa as configurações da viagem         | Altera os dados e salva          | O sistema atualiza as informações da viagem               |
| Atualização do objetivo da viagem     | O Organizador acessa o formulário                       | Informa um novo objetivo e salva | O sistema registra a alteração                            |
| Alteração das datas                   | O Organizador informa novas datas válidas               | Salva as alterações              | O sistema atualiza o período da viagem                    |
| Data final inválida                   | O Organizador informa uma data final anterior à inicial | Salva as alterações              | O sistema informa que o período é inválido                |
| Upload de imagem                      | O Organizador seleciona uma imagem válida               | Confirma o envio                 | O sistema associa a imagem à viagem                       |
| Participante tenta editar informações | Um participante acessa a viagem                         | Tenta alterar os dados básicos   | O sistema bloqueia a operação                             |
| Falha ao salvar alterações            | O Organizador realiza modificações válidas              | Ocorre erro durante o salvamento | O sistema informa que não foi possível atualizar a viagem |

## Observações

1. As informações cadastradas devem ficar visíveis para todos os participantes da viagem.

2. A imagem da viagem será utilizada para facilitar a identificação visual da viagem dentro da plataforma.

3. Apenas o Organizador pode alterar as informações iniciais da viagem.

4. As informações cadastradas poderão ser utilizadas posteriormente pelos mecanismos de recomendação e geração de roteiros da plataforma.

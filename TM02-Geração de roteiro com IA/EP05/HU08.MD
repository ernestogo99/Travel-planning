## Conversa

Durante o planejamento da viagem, a IA deve analisar as interações realizadas pelos participantes, como sugestões cadastradas, avaliações positivas e negativas e histórico de viagens. Com base nessas informações, a IA deve identificar os interesses predominantes do grupo e apresentá-los ao Organizador. Essas informações servirão como base para a geração de recomendações e roteiros personalizados.

## Critérios de Aceitação

| #   | Critério                                                                                       |
| --- | ---------------------------------------------------------------------------------------------- |
| 1   | O sistema deve permitir ao Organizador visualizar os interesses identificados pela IA          |
| 2   | A IA deve considerar as avaliações realizadas pelos participantes                              |
| 3   | A IA deve considerar as sugestões cadastradas pelos participantes                              |
| 4   | A IA deve considerar informações históricas disponíveis sobre viagens anteriores               |
| 5   | O sistema deve apresentar os interesses identificados de forma compreensível                   |
| 6   | O sistema deve exibir a data da última análise realizada                                       |
| 7   | O sistema deve atualizar os interesses identificados quando novas interações forem registradas |
| 8   | Apenas participantes vinculados à viagem podem contribuir para a análise do perfil do grupo    |
| 9   | Apenas o Organizador pode acessar a visão consolidada dos interesses do grupo                  |
| 10  | O sistema deve informar quando não existirem dados suficientes para realizar a análise         |

## Cenários de Teste

| Cenário                                  | Dado                                              | Quando                                 | Então                                                       |
| ---------------------------------------- | ------------------------------------------------- | -------------------------------------- | ----------------------------------------------------------- |
| Visualização de interesses identificados | Existem interações suficientes registradas        | O Organizador acessa a funcionalidade  | O sistema exibe os interesses identificados pela IA         |
| Atualização do perfil do grupo           | Novas avaliações ou sugestões são registradas     | A IA realiza nova análise              | O sistema atualiza os interesses apresentados               |
| Dados insuficientes                      | A viagem possui poucas interações registradas     | O Organizador acessa a funcionalidade  | O sistema informa que não há dados suficientes para análise |
| Acesso autorizado                        | O Organizador acessa a funcionalidade             | Solicita a visualização dos interesses | O sistema exibe o perfil identificado                       |
| Acesso não autorizado                    | Um participante tenta acessar a visão consolidada | Solicita a visualização                | O sistema impede o acesso                                   |
| Falha na análise                         | Ocorre erro durante o processamento da IA         | O Organizador solicita a visualização  | O sistema informa que não foi possível concluir a análise   |

## Observações

1. A identificação de interesses deve considerar as interações realizadas pelos participantes durante o planejamento.

2. Os interesses identificados servem como apoio para geração de recomendações e roteiros personalizados.

3. A análise realizada pela IA não substitui as decisões do grupo, apenas auxilia o processo de planejamento.

4. O sistema pode utilizar avaliações, sugestões e histórico de viagens para compor o perfil identificado.

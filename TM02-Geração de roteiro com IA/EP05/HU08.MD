## Conversa

A plataforma deve utilizar Inteligência Artificial para analisar as interações realizadas pelos participantes da viagem, como sugestões cadastradas, avaliações positivas e negativas e histórico de atividades. Com base nessas informações, a IA deve identificar os interesses predominantes do grupo e apresentá-los ao Organizador da viagem. Essas informações servirão de apoio para a geração de recomendações e roteiros personalizados.

## Critérios de Aceitação

| #   | Critério                                                                                       |
| --- | ---------------------------------------------------------------------------------------------- |
| 1   | O sistema deve permitir ao Organizador visualizar os interesses identificados pela IA          |
| 2   | A IA deve analisar as sugestões cadastradas pelos participantes                                |
| 3   | A IA deve analisar as avaliações realizadas pelos participantes                                |
| 4   | A IA deve considerar informações históricas disponíveis da viagem                              |
| 5   | O sistema deve apresentar os interesses identificados de forma compreensível                   |
| 6   | O sistema deve exibir a data da última análise realizada                                       |
| 7   | O sistema deve atualizar os interesses identificados quando novas interações forem registradas |
| 8   | Apenas o Organizador da viagem pode visualizar a análise consolidada                           |
| 9   | O sistema deve informar quando não existirem dados suficientes para identificar interesses     |
| 10  | O sistema deve manter os dados utilizados na análise vinculados à viagem correspondente        |

## Cenários de Teste

| Cenário                                   | Dado                                                 | Quando                                 | Então                                                     |
| ----------------------------------------- | ---------------------------------------------------- | -------------------------------------- | --------------------------------------------------------- |
| Visualização dos interesses identificados | Existem interações suficientes registradas na viagem | O Organizador acessa a funcionalidade  | O sistema apresenta os interesses identificados pela IA   |
| Atualização dos interesses                | Novas avaliações ou sugestões são registradas        | A IA executa nova análise              | O sistema atualiza os interesses exibidos                 |
| Dados insuficientes                       | A viagem possui poucas interações registradas        | O Organizador solicita a análise       | O sistema informa que não há dados suficientes            |
| Acesso autorizado                         | O Organizador acessa a funcionalidade                | Solicita a visualização dos interesses | O sistema exibe a análise realizada                       |
| Acesso não autorizado                     | Um participante tenta acessar a análise consolidada  | Solicita a visualização                | O sistema impede o acesso                                 |
| Falha na análise                          | Ocorre erro durante o processamento da IA            | O Organizador solicita a visualização  | O sistema informa que não foi possível concluir a análise |

## Observações

1. Os interesses identificados representam tendências observadas nas interações dos participantes.

2. A análise realizada pela IA tem caráter de apoio à decisão e não substitui as escolhas do grupo.

3. Os interesses identificados podem ser utilizados posteriormente na geração de recomendações e roteiros personalizados.

4. A funcionalidade deve seguir os princípios de Inteligência Artificial Confiável e Centrada no Ser Humano, fornecendo informações que auxiliem o Organizador na tomada de decisão.

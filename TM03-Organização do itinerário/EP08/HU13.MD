## Cartão

| Elemento | Descrição                                                             |
| -------- | --------------------------------------------------------------------- |
| Como     | Organizador da viagem                                                 |
| Quero    | Adicionar uma atividade ao cronograma da viagem                       |
| Para que | Os participantes possam visualizar e acompanhar os eventos planejados |

## Conversa

O Organizador da viagem deve ser capaz de adicionar atividades ao cronograma para estruturar o planejamento da viagem. Durante o cadastro, devem ser informadas as principais informações da atividade, como nome, descrição, local, data, horário e categoria. Após a criação, a atividade deve ficar disponível no cronograma da viagem correspondente, permitindo que os participantes visualizem seus detalhes e acompanhem a programação planejada.

## Critérios de Aceitação

| #   | Critério                                                                                |
| --- | --------------------------------------------------------------------------------------- |
| 1   | O sistema deve disponibilizar uma opção para adicionar uma nova atividade ao cronograma |
| 2   | Apenas o Organizador da viagem pode criar atividades                                    |
| 3   | O sistema deve exibir um formulário para cadastro da atividade                          |
| 4   | O formulário deve permitir informar o nome da atividade                                 |
| 5   | O formulário deve permitir informar uma descrição da atividade                          |
| 6   | O formulário deve permitir informar a localização da atividade                          |
| 7   | O formulário deve permitir informar a data da atividade                                 |
| 8   | O formulário deve permitir informar o horário de início da atividade                    |
| 9   | O formulário deve permitir informar o horário de término da atividade                   |
| 10  | O formulário deve permitir definir a categoria da atividade                             |
| 11  | O sistema deve permitir marcar a atividade como marco da viagem                         |
| 12  | O sistema deve validar o preenchimento dos campos obrigatórios                          |
| 13  | O sistema deve validar que o horário de término seja posterior ao horário de início     |
| 14  | O sistema deve validar que a data da atividade esteja dentro do período da viagem       |
| 15  | O sistema deve associar automaticamente a atividade à viagem selecionada                |
| 16  | O sistema deve armazenar a atividade após a confirmação do cadastro                     |
| 17  | O sistema deve exibir a atividade no cronograma da viagem após sua criação              |
| 18  | O sistema deve exibir uma mensagem de confirmação após a criação da atividade           |

## Cenários de Teste

| Cenário                                | Dado                                                    | Quando                                                      | Então                                                                      |
| -------------------------------------- | ------------------------------------------------------- | ----------------------------------------------------------- | -------------------------------------------------------------------------- |
| Cadastro válido de atividade           | O Organizador acessa o cronograma da viagem             | Preenche todos os campos obrigatórios e confirma o cadastro | O sistema cria a atividade e a exibe no cronograma                         |
| Nome não informado                     | O Organizador acessa o formulário de cadastro           | Deixa o nome da atividade vazio e tenta salvar              | O sistema informa que o nome é obrigatório                                 |
| Localização não informada              | O Organizador preenche parcialmente o formulário        | Não informa a localização e tenta salvar                    | O sistema informa que a localização é obrigatória                          |
| Horário inválido                       | O Organizador informa horário final anterior ao inicial | Confirma o cadastro                                         | O sistema informa que o horário é inválido                                 |
| Data fora do período da viagem         | O Organizador informa uma data fora da viagem           | Confirma o cadastro                                         | O sistema informa que a atividade deve ocorrer dentro do período da viagem |
| Atividade marcada como marco da viagem | O Organizador seleciona a opção de marco da viagem      | Finaliza o cadastro                                         | O sistema identifica a atividade como marco da viagem                      |
| Cancelamento do cadastro               | O Organizador inicia o cadastro                         | Seleciona cancelar                                          | O sistema fecha o formulário sem salvar a atividade                        |
| Falha ao salvar atividade              | O Organizador informa dados válidos                     | Ocorre erro durante o armazenamento                         | O sistema informa que não foi possível criar a atividade                   |

## Observações

1. Uma atividade representa um evento planejado dentro da viagem.

2. Exemplos de atividades:
   - Visitar ponto turístico;
   - Check-in em hotel;
   - Restaurante;
   - Evento cultural;
   - Deslocamento;
   - Marco da viagem.

3. Apenas usuários com papel de Organizador podem criar atividades.

4. Toda atividade deve estar vinculada a uma viagem existente.

5. As atividades devem ser exibidas agrupadas por dia no cronograma da viagem.

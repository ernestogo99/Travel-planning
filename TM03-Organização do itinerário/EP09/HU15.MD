## Cartão

| Elemento | Descrição                                                                                     |
| -------- | --------------------------------------------------------------------------------------------- |
| Como     | Participante da viagem                                                                        |
| Quero    | Visualizar os detalhes de uma atividade e interagir com ela                                   |
| Para que | Eu possa acompanhar as informações da atividade e confirmar minha participação quando desejar |

## Conversa

Os usuários da viagem devem poder acessar os detalhes de uma atividade cadastrada no cronograma. A tela de detalhes deve apresentar informações completas da atividade, como nome, descrição, localização, data, horário e participantes confirmados. Dependendo do perfil do usuário, ações específicas devem estar disponíveis. Participantes podem confirmar presença na atividade, enquanto Organizadores podem editar suas informações para manter o planejamento atualizado.

## Critérios de Aceitação

| #   | Critério                                                                                               |
| --- | ------------------------------------------------------------------------------------------------------ |
| 1   | O sistema deve permitir visualizar os detalhes de uma atividade selecionada                            |
| 2   | O sistema deve exibir o nome da atividade                                                              |
| 3   | O sistema deve exibir a descrição da atividade                                                         |
| 4   | O sistema deve exibir a localização da atividade                                                       |
| 5   | O sistema deve exibir a data da atividade                                                              |
| 6   | O sistema deve exibir o horário de início e término da atividade                                       |
| 7   | O sistema deve exibir a lista de participantes confirmados                                             |
| 8   | O sistema deve exibir a quantidade de participantes confirmados                                        |
| 9   | Participantes da viagem devem poder confirmar presença na atividade                                    |
| 10  | O sistema deve registrar a participação do usuário na atividade                                        |
| 11  | O sistema deve atualizar a quantidade de participantes após uma confirmação                            |
| 12  | O sistema deve impedir confirmações duplicadas para a mesma atividade                                  |
| 13  | Organizadores da viagem devem visualizar a opção de editar atividade                                   |
| 14  | O sistema deve permitir ao Organizador alterar as informações da atividade                             |
| 15  | As alterações realizadas pelo Organizador devem ser refletidas imediatamente nos detalhes da atividade |
| 16  | Apenas usuários vinculados à viagem podem acessar os detalhes da atividade                             |
| 17  | O sistema deve informar quando ocorrer falha ao registrar participação ou salvar alterações            |

## Cenários de Teste

| Cenário                           | Dado                                                          | Quando                                      | Então                                                      |
| --------------------------------- | ------------------------------------------------------------- | ------------------------------------------- | ---------------------------------------------------------- |
| Visualização dos detalhes         | Existe uma atividade cadastrada                               | O usuário acessa a atividade                | O sistema exibe suas informações completas                 |
| Confirmação de participação       | O participante ainda não confirmou presença                   | Seleciona a opção participar                | O sistema registra sua participação                        |
| Atualização de participantes      | Um participante confirma presença                             | A confirmação é concluída                   | O sistema atualiza a lista e a quantidade de participantes |
| Participação duplicada            | O participante já confirmou presença                          | Tenta participar novamente                  | O sistema impede o registro duplicado                      |
| Edição da atividade               | O Organizador acessa os detalhes da atividade                 | Seleciona editar e salva alterações válidas | O sistema atualiza as informações da atividade             |
| Acesso de participante            | Um participante acessa a atividade                            | Visualiza os detalhes                       | O sistema exibe o botão participar                         |
| Acesso de organizador             | O Organizador acessa a atividade                              | Visualiza os detalhes                       | O sistema exibe o botão editar                             |
| Usuário não autorizado            | Um usuário sem vínculo com a viagem tenta acessar a atividade | Solicita visualização                       | O sistema impede o acesso                                  |
| Falha no registro de participação | Ocorre erro durante o processamento                           | O participante tenta participar             | O sistema informa que não foi possível concluir a operação |

## Observações

1. A tela de detalhes da atividade deve adaptar as ações disponíveis conforme o papel do usuário na viagem.

2. Participantes podem confirmar presença em atividades, mas não podem alterar suas informações.

3. Organizadores podem editar atividades para manter o cronograma atualizado.

4. A confirmação de participação deve ser refletida para todos os usuários da viagem.

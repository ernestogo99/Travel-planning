| Para que | Eu possa manter o planejamento atualizado e remover atividades que não farão mais parte da viagem |

## Conversa

Durante o planejamento da viagem, pode ser necessário remover atividades que foram canceladas, substituídas ou que não fazem mais sentido para o grupo. O Organizador da viagem deve poder excluir atividades previamente cadastradas no cronograma. Após a confirmação da exclusão, a atividade não deve mais ser exibida aos participantes nem considerada no planejamento da viagem.

## Critérios de Aceitação

| #   | Critério                                                                            |
| --- | ----------------------------------------------------------------------------------- |
| 1   | Apenas o Organizador da viagem pode remover atividades                              |
| 2   | O sistema deve disponibilizar uma opção para remoção da atividade                   |
| 3   | O sistema deve solicitar confirmação antes da exclusão da atividade                 |
| 4   | O sistema deve exibir as informações da atividade antes da confirmação da exclusão  |
| 5   | O sistema deve remover a atividade do cronograma após a confirmação                 |
| 6   | O sistema não deve exibir a atividade removida no cronograma da viagem              |
| 7   | O sistema deve remover os registros de participação associados à atividade excluída |
| 8   | O sistema deve atualizar automaticamente o cronograma após a exclusão               |
| 9   | O sistema deve exibir mensagem de sucesso após a remoção da atividade               |
| 10  | O sistema deve impedir que participantes removam atividades                         |
| 11  | O sistema deve informar quando ocorrer falha durante a exclusão                     |
| 12  | A remoção da atividade deve ser refletida para todos os participantes da viagem     |

## Cenários de Teste

| Cenário                                 | Dado                                            | Quando                              | Então                                                         |
| --------------------------------------- | ----------------------------------------------- | ----------------------------------- | ------------------------------------------------------------- |
| Remoção bem-sucedida                    | Existe uma atividade cadastrada no cronograma   | O Organizador confirma a exclusão   | O sistema remove a atividade e atualiza o cronograma          |
| Cancelamento da remoção                 | O Organizador inicia a exclusão da atividade    | Seleciona cancelar                  | O sistema mantém a atividade sem alterações                   |
| Participante tenta remover atividade    | Um participante acessa os detalhes da atividade | Tenta realizar a exclusão           | O sistema impede a operação                                   |
| Atualização do cronograma               | Uma atividade é removida com sucesso            | O cronograma é recarregado          | O sistema não exibe mais a atividade removida                 |
| Falha na exclusão                       | O Organizador confirma a exclusão               | Ocorre erro durante o processamento | O sistema informa que não foi possível remover a atividade    |
| Atividade com participantes confirmados | Existem participantes registrados na atividade  | O Organizador remove a atividade    | O sistema remove a atividade e seus registros de participação |

## Observações

1. Apenas usuários com papel de Organizador podem remover atividades.

2. A exclusão de uma atividade afeta todos os participantes da viagem.

3. Após a remoção, a atividade deixa de aparecer no cronograma e em qualquer visualização relacionada.

4. Os registros de participação associados à atividade removida devem ser excluídos juntamente com a atividade.

## Conversa

Durante o planejamento e execução da viagem, os participantes precisam ser informados sobre acontecimentos relevantes relacionados ao cronograma. O sistema deve gerar notificações automaticamente quando ocorrerem eventos importantes, como criação, alteração ou remoção de atividades, além de lembretes sobre atividades próximas. Essas notificações devem ficar disponíveis em uma central de notificações para consulta pelos participantes da viagem.

## Critérios de Aceitação

| #   | Critério                                                                                       |
| --- | ---------------------------------------------------------------------------------------------- |
| 1   | O sistema deve gerar notificações quando uma atividade for criada                              |
| 2   | O sistema deve gerar notificações quando uma atividade for alterada                            |
| 3   | O sistema deve gerar notificações quando uma atividade for removida                            |
| 4   | O sistema deve gerar notificações para atividades próximas de seu horário de início            |
| 5   | O sistema deve disponibilizar uma central de notificações para os participantes da viagem      |
| 6   | Cada notificação deve possuir título e descrição                                               |
| 7   | Cada notificação deve possuir data e horário de geração                                        |
| 8   | As notificações devem estar associadas à viagem correspondente                                 |
| 9   | O sistema deve armazenar as notificações para consulta posterior                               |
| 10  | O sistema deve exibir as notificações em ordem cronológica, da mais recente para a mais antiga |
| 11  | O participante deve poder visualizar o histórico de notificações recebidas                     |
| 12  | Apenas participantes da viagem devem visualizar notificações relacionadas àquela viagem        |
| 13  | O sistema deve informar quando não existirem notificações disponíveis                          |
| 14  | O sistema deve atualizar a lista de notificações quando novos eventos forem gerados            |
| 15  | O sistema deve informar caso ocorra falha na geração ou recuperação das notificações           |

## Cenários de Teste

| Cenário                      | Dado                                                          | Quando                                          | Então                                                 |
| ---------------------------- | ------------------------------------------------------------- | ----------------------------------------------- | ----------------------------------------------------- |
| Criação de atividade         | O Organizador cria uma nova atividade                         | A atividade é salva com sucesso                 | O sistema gera uma notificação para os participantes  |
| Alteração de atividade       | Existe uma atividade cadastrada                               | O Organizador altera suas informações           | O sistema gera uma notificação informando a alteração |
| Remoção de atividade         | Existe uma atividade cadastrada                               | O Organizador remove a atividade                | O sistema gera uma notificação informando a remoção   |
| Atividade próxima            | Existe uma atividade com horário próximo                      | O momento configurado para aviso é alcançado    | O sistema gera uma notificação de lembrete            |
| Visualização de notificações | Existem notificações registradas                              | O participante acessa a central de notificações | O sistema exibe as notificações disponíveis           |
| Histórico de notificações    | Existem notificações antigas registradas                      | O participante consulta a central               | O sistema apresenta o histórico em ordem cronológica  |
| Ausência de notificações     | Não existem notificações registradas                          | O participante acessa a central                 | O sistema informa que não há notificações disponíveis |
| Usuário não participante     | Um usuário sem vínculo com a viagem acessa a central          | Solicita visualização das notificações          | O sistema impede o acesso                             |
| Falha no processamento       | Ocorre erro durante a geração ou recuperação das notificações | O participante acessa a funcionalidade          | O sistema informa a indisponibilidade da operação     |

## Observações

1. As notificações devem ser geradas automaticamente pelo sistema.

2. Uma notificação pode estar relacionada à criação, alteração, remoção ou proximidade de uma atividade.

3. As notificações devem ser vinculadas à viagem responsável por sua geração.

4. O objetivo da funcionalidade é manter os participantes informados sobre eventos relevantes do cronograma.

5. A central de notificações deve apresentar apenas informações relacionadas às viagens das quais o usuário participa.

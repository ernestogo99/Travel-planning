# HU17 - Gerar notificações da viagem

## Cartão

| Elemento | Descrição                                                                  |
| -------- | -------------------------------------------------------------------------- |
| Como     | Organizador da viagem                                                      |
| Quero    | Que o sistema gere notificações relacionadas às atividades da viagem       |
| Para que | Os participantes sejam informados sobre eventos e alterações do cronograma |

## Conversa

Durante o planejamento e execução da viagem, os participantes precisam ser informados sobre atividades cadastradas, alterações realizadas no cronograma e eventos próximos de acontecer. O sistema deve gerar notificações automaticamente a partir das ações realizadas pelos Organizadores e das informações registradas nas atividades da viagem. Essas notificações devem ficar disponíveis para consulta pelos participantes.

## Critérios de Aceitação

| #   | Critério                                                                                       |
| --- | ---------------------------------------------------------------------------------------------- |
| 1   | O sistema deve gerar notificações quando uma nova atividade for criada                         |
| 2   | O sistema deve gerar notificações quando uma atividade for alterada                            |
| 3   | O sistema deve gerar notificações quando uma atividade for removida                            |
| 4   | O sistema deve gerar notificações para atividades próximas de seu horário de início            |
| 5   | Cada notificação deve possuir título e descrição                                               |
| 6   | Cada notificação deve possuir data e horário de geração                                        |
| 7   | As notificações devem estar associadas à viagem correspondente                                 |
| 8   | O sistema deve registrar o tipo da notificação gerada                                          |
| 9   | Apenas participantes da viagem devem receber notificações relacionadas àquela viagem           |
| 10  | O sistema deve armazenar as notificações geradas para consulta posterior                       |
| 11  | O sistema deve exibir as notificações em ordem cronológica, da mais recente para a mais antiga |
| 12  | O sistema deve informar caso ocorra falha durante a geração da notificação                     |

## Cenários de Teste

| Cenário                | Dado                                     | Quando                                           | Então                                                                      |
| ---------------------- | ---------------------------------------- | ------------------------------------------------ | -------------------------------------------------------------------------- |
| Criação de atividade   | O Organizador cria uma nova atividade    | A atividade é salva com sucesso                  | O sistema gera uma notificação para os participantes                       |
| Alteração de atividade | Existe uma atividade cadastrada          | O Organizador altera suas informações            | O sistema gera uma notificação informando a alteração                      |
| Remoção de atividade   | Existe uma atividade cadastrada          | O Organizador remove a atividade                 | O sistema gera uma notificação informando a remoção                        |
| Atividade próxima      | Existe uma atividade com horário próximo | O momento configurado para aviso é alcançado     | O sistema gera uma notificação de lembrete                                 |
| Consulta posterior     | Existem notificações registradas         | O participante acessa a central de notificações  | O sistema exibe as notificações armazenadas                                |
| Falha na geração       | Ocorre erro durante o processamento      | Um evento que deveria gerar notificação acontece | O sistema registra a falha e informa a indisponibilidade da funcionalidade |

## Observações

1. As notificações devem ser geradas automaticamente pelo sistema.

2. O Organizador não precisa criar notificações manualmente.

3. Uma notificação pode estar relacionada à criação, alteração, remoção ou proximidade de uma atividade.

4. As notificações devem ser vinculadas à viagem responsável por sua geração.

5. O objetivo da funcionalidade é manter os participantes informados sobre eventos relevantes do cronograma.

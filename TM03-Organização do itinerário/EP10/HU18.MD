## Conversa

Os participantes da viagem devem ser informados sobre acontecimentos relevantes relacionados ao planejamento e execução da viagem. Para isso, o sistema deve disponibilizar uma central de notificações contendo avisos gerados automaticamente a partir de eventos da viagem, como criação, alteração ou remoção de atividades, além de lembretes sobre compromissos próximos. As notificações devem permanecer disponíveis para consulta pelos participantes.

## Critérios de Aceitação

| #   | Critério                                                                                       |
| --- | ---------------------------------------------------------------------------------------------- |
| 1   | O sistema deve disponibilizar uma área de notificações para os participantes da viagem         |
| 2   | O sistema deve exibir notificações relacionadas às viagens das quais o usuário participa       |
| 3   | Cada notificação deve apresentar título e descrição                                            |
| 4   | Cada notificação deve apresentar data e horário de geração                                     |
| 5   | O sistema deve exibir as notificações em ordem cronológica, da mais recente para a mais antiga |
| 6   | O participante deve poder visualizar o histórico de notificações recebidas                     |
| 7   | O sistema deve exibir notificações de criação de atividades                                    |
| 8   | O sistema deve exibir notificações de alteração de atividades                                  |
| 9   | O sistema deve exibir notificações de remoção de atividades                                    |
| 10  | O sistema deve exibir notificações de lembretes de atividades próximas                         |
| 11  | Apenas participantes da viagem devem visualizar notificações relacionadas àquela viagem        |
| 12  | O sistema deve informar quando não existirem notificações disponíveis                          |
| 13  | O sistema deve atualizar a lista de notificações quando novos eventos forem gerados            |

## Cenários de Teste

| Cenário                      | Dado                                                 | Quando                                          | Então                                                 |
| ---------------------------- | ---------------------------------------------------- | ----------------------------------------------- | ----------------------------------------------------- |
| Visualização de notificações | Existem notificações registradas para a viagem       | O participante acessa a central de notificações | O sistema exibe as notificações disponíveis           |
| Histórico de notificações    | Existem notificações antigas registradas             | O participante consulta a central               | O sistema apresenta o histórico em ordem cronológica  |
| Nova notificação gerada      | Uma atividade é alterada pelo Organizador            | O participante acessa a central de notificações | O sistema exibe a nova notificação                    |
| Ausência de notificações     | Não existem notificações registradas                 | O participante acessa a central                 | O sistema informa que não há notificações disponíveis |
| Usuário não participante     | Um usuário sem vínculo com a viagem acessa a central | Solicita visualização das notificações          | O sistema impede o acesso                             |
| Atualização da lista         | Novas notificações são geradas                       | O participante atualiza a tela                  | O sistema exibe as notificações mais recentes         |

## Observações

1. A central de notificações deve apresentar apenas informações relacionadas às viagens das quais o usuário participa.

2. O objetivo da funcionalidade é manter os participantes informados sobre eventos relevantes da viagem.
